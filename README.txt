    Як запустити програму?
---------------------------------------
1.Вкажіть у config.py наступні налаштування
    * YEARS - роки по яких необхідно завантажити дані та згенерувати статистику
    * STATISTIC_CSV - файл у якому буде збережена статистика
    * DB_NAME - назва БД
    * DB_USER - користувач БД
    * DB_PASS- пароль користувача БД
    * DB_PORT - порт БД
    * DB_HOST - хост БД
2. Запустіть програму
>> python3 main.py

    За що відповідають файли?
---------------------------------------
READMY.txt - інформаційний файл
requiremenst.txt - список залежностей
statistic.csv - файл що містить зібрану статистику згідно варіанту 1
schema.sql - структура таблиці що буде створена в БД
main.py - основний файл для запуску
app/config.py - конфігураційний файл
app/db.py - файл що відоповідає за роботу із БД
app/downloader.py - файл що відповідає за роботу із мережею та скачування даних
app/extracter.py - файл що відповідає за розпакування скачаних архівів
                   та приведення файлів до необхідного формату

        statistic.csv
---------------------------------------
RegName - назва регіону
Year - рік ЗНО
MaxUkrBal100 - максимальний бал із ЗНО з української мови та літератури

        Відновлення роботи програми
---------------------------------------
1. Відновлення роботи програми у випадку втрати 
або неможливості встановити з'єднання із мережею

    * Перед запуском програми відключіть з'єднання із мережею.
    * І програма буде роботи що 5 с спроби з'єднатись до того часу поки ще не відбудеться.
    * Включіть з'єдання до мережі
    * Програма продовжить скачувати файли

2. Відновлення роботи програми у випадку 
неможливості встановити з'єднання із БД
    * Перед запуском програми змініть пароль для користувача у pgAdmin на відміний від паролю у config.py.
    * Або запустіть програму нестворивши бази даних що вказана у config.py
    * Програма буде намагатись що 5 с з'єднатись із БД
    * Встановіть в pgAdmin вірний пароль
    * Або створіть необхідну БД
    * Програма зможе встановити тоді з'єднання та продовжить свою роботу

3. Відновлення роботи програми у збої в роботі БД
(дані будуть відкоченні назад завдаки транзакціям)
    * встановіть у config.py - SIMULATE_ERROR у значення True
    * в тому випадку функція (extracter.py 38-44)
      допише у кінець одного із csv файлів некоректні дані що не можуть бути збереженні у БД
    * коли дойде черга до запису із цього файла у БД - виникне виняток і програма відкотить цю транзакцію до початку даного файлу
    * слідуючі/попередні файли з даними запишуться без жодних проблем

Унікальність записів у БД гарантується через PRIMARY KEY - UUID
